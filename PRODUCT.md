# Product

## Register

product

## Users

OpenClaw operators, admins, and support staff who manage ERP chatbot agents on customer servers. They use the app during setup, incident response, routine monitoring, channel configuration, and customer support without needing to SSH into the server for common tasks.

## Product Purpose

OpenClaw Admin is an operational console for configuring and monitoring OpenClaw agents, model credentials, Telegram/LINE/WebChat channels, MCP access, gateway state, sessions, memory, orders, and system health. Success means operators can deploy, diagnose, and recover production chatbot systems safely, with clear status and reversible actions.

## Brand Personality

Operational, trustworthy, restrained. The interface should feel like a dependable control room: dense enough for repeated admin work, calm under incident pressure, and explicit about risk.

## Anti-references

Avoid decorative SaaS marketing dashboards, oversized hero layouts, unclear one-click destructive actions, hidden side effects, secret exposure, and UI patterns that make a risky server mutation feel casual.

## Design Principles

- Show system truth before action.
- Make risky operations reversible and auditable.
- Keep operator workflows dense, scannable, and calm.
- Prefer explicit status, next steps, and recovery commands over vague errors.
- Preserve existing production behavior unless a change is clearly safer.

## Accessibility & Inclusion

Target practical WCAG 2.2 AA for admin workflows: readable contrast, keyboard-usable controls, descriptive button labels, clear focus states, and reduced-motion-friendly state changes.
